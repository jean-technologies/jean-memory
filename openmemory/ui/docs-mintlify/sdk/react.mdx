## Installation

Install the Jean Memory React SDK:

```bash
npm install @jeanmemory/react
```

<Warning>
**Account Requirement**: This SDK only works with Jean Memory accounts created using email/password. Accounts created via Google or GitHub sign-in are not currently supported.
</Warning>

## How It Works - MCP Integration

The Jean Memory React SDK uses **Model Context Protocol (MCP)** to provide truly intelligent, personalized AI responses:

1. **User signs in** with their Jean Memory account
2. **MCP jean_memory tool** automatically retrieves personal context about the user
3. **AI synthesis** generates natural responses using the user's context + your system prompt
4. **Conversations are stored** back to Jean Memory for future context

This means your AI **actually knows the user** - not just generic responses!

## Advanced Usage with `useJeanAgent` Hook

If you need custom UI or more control, use the hook directly:

```jsx
import { useJeanAgent, SignInWithJean, JeanChat } from '@jeanmemory/react';

function CustomApp() {
  const agent = useJeanAgent({
    apiKey: "jean_sk_your_api_key_here",
    systemPrompt: "You are a helpful assistant"
  });

  if (!agent.isAuthenticated) {
    return <SignInWithJean onSuccess={agent.signIn} />;
  }

  return <JeanChat agent={agent} />;
}
```

### useJeanAgent Returns

- **user**: Current user (`JeanUser | null`)
- **isLoading**: Authentication loading state
- **error**: Error message if any
- **messages**: Conversation history
- **isAuthenticated**: Boolean for auth state
- **signIn(email, password)**: Sign in function
- **sendMessage(message)**: Send message to AI
- **storeDocument(title, content)**: Store document to Jean Memory
- **clearConversation()**: Clear current chat

## Under The Hood - MCP Architecture

Jean Memory React SDK uses **Model Context Protocol (MCP)** for intelligent personalization:

1. **Sign In**: User authenticates with Jean Memory account
2. **Context Retrieval**: `jean_memory` MCP tool fetches user's personal context
3. **AI Synthesis**: Server-side LLM generates responses using context + system prompt
4. **Memory Storage**: `store_document` MCP tool saves conversations back to Jean Memory
5. **Continuous Learning**: Each interaction improves future personalization

This architecture ensures your AI actually **knows and remembers** each user!

## Development Setup

For local development, add a proxy to your `package.json` to avoid CORS issues:

```json
{
  "name": "my-app",
  "scripts": { ... },
  "proxy": "https://jean-memory-api-virginia.onrender.com"
}
```

Then restart your development server with `npm start`.

## Troubleshooting

### Common Issues

**"API key validation failed"**
- Ensure your API key starts with `jean_sk_`
- Get a valid API key from [jeanmemory.com](https://jeanmemory.com)
- Check that your API key is not expired

**"Authentication failed"**
- Verify you have a Jean Memory account created with email/password at [jeanmemory.com](https://jeanmemory.com)
- ‚ö†Ô∏è Google/GitHub accounts are not supported - create a new account with email/password
- Check your email and password are correct
- Ensure your account is active

**"MCP request failed"**
- Check browser network tab for specific error details
- Verify your API key has MCP tool permissions
- Try refreshing and signing in again

**CORS errors (Access Control Allow Origin)**
- Add proxy to `package.json` for development (see Development Setup above)
- For production, contact Jean Memory team to whitelist your domain
- Ensure you're using the correct API endpoint

**No AI responses**
- Check browser console for errors
- Ensure the `systemPrompt` is provided
- Verify the jean_memory MCP tool is working

### Getting Help

- **Account Setup**: Visit [jeanmemory.com](https://jeanmemory.com)
- **Technical Issues**: Check browser console for detailed error messages
- **API Problems**: Verify your API key permissions and limits
